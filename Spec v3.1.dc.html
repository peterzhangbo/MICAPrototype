需求对照 · FR-004
用途
邮箱注册并在 coinw.eu 侧完成验证，再创建 ZBX 用户。
组件 (DS)
单列容器（约 480）· Input For Verify（邮箱/OTP）· 分项同意（2 个复选框）：① ZBX 用户协议 + 隐私政策 ② 风险披露，链接品牌色 #5227FF / 加粗 → S11；默认不勾选（GDPR 不可预勾）。Sign Up Primary Brand（两项均勾选前置灰 / disabled；loading 态）。
五态

数据 / 接口

边界 / 约束
传入 email 必须是 coinw.eu 已验证邮箱，不可为用户随意填写的未验证邮箱。邮箱已存在 → 引导去登录（不在注册态报错）。注册按钮在 两项同意均勾选前置灰；同意记录由 ZBX 存储，前端不落库（不新增数据模型字段）。Cookie 不在注册同意内（见 Cookie 同意 CMP）。
验收
注册成功获得 uid；邮箱已在 coinw.eu 侧完成验证。
⟳ CR-2026-001 · 分项勾选
3 项同意拆为 2 个复选框：① 用户协议+隐私 ② 风险披露，两项均必选。a11y：每框 aria-required，未勾提交 aria-invalid + aria-describedby，Space 可勾。文案占位、待法务定稿，不得伪造监管编号。

需求对照 · Cookie 决策（§4 / CR-2026-001）
用途
在非必要 Cookie / 追踪触发前取得同意。经核实实际 Cookie 清单非「严格必要」（注意 Sumsub iframe、GeoIP、分析/监控脚本），故须部署 CMP。
原则（GDPR/ePrivacy）
与注册同意分离 · 可拒绝（Reject 与 Accept 等权同层）· 可撤回（页脚常驻入口）· 按类别授权 · 非必要默认关，选择前不触发。
组件 (DS)
底部 Banner（Reject all / Manage / Accept all）· 偏好面板（按类别 Toggle：严格必要[锁定开] · 功能 · 分析）· 页脚「Cookie Settings」常驻入口。
边界 / 约束
严格必要 Cookie 始终开、不可关；Cookie 同意不放进注册 3 项同意；与 Art.78(5) 订单路由同意无关（MVP 不做）。
⏳ 待法务
实际 Cookie 清单、类别划分与文案待法务定稿；上线前核对清单确认分类无误，不得伪造。
验收
非必要 Cookie 在同意前不触发；可拒绝 / 可撤回 / 按类别；页脚有 Cookie Settings 入口；与注册同意分离。
